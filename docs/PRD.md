# 공공직군 행정업무 슈퍼앱 PRD

## 1. 문서 목적

공공직군 행정업무 슈퍼앱을 개발하기 전, 핵심 기능의 사용자 문제, 요구사항, 우선순위, 성공 기준을 정리한다. 본 문서는 기획, 개발, 테스트, 운영 담당자가 동일한 범위와 품질 기준으로 협업하기 위한 기준 문서다.

## 2. 제품 개요

공공직군 행정업무 슈퍼앱은 팀 단위 행정업무에서 반복적으로 발생하는 일정 공유, 엑셀 정리, 민원 응대, 행정 뉴스 모니터링을 하나의 업무 화면에서 처리하는 내부 업무 지원 도구다.

주요 사용자는 공공기관, 지자체, 공공위탁 조직의 실무자, 팀장, 행정지원 담당자다.

## 3. 목표

- 팀원의 휴가, 근무, 출장 일정을 한 화면에서 공유한다.
- 반복적인 엑셀 파일 분리/병합 업무 시간을 줄인다.
- 민원 매뉴얼 기반으로 일관된 응대 스크립트를 생성한다.
- 공공행정 관련 뉴스를 매일 아침 자동 수집해 업무 참고자료로 제공한다.
- 개인정보, 내부문서, 민원정보를 안전하게 다룰 수 있는 운영 기준을 마련한다.

## 4. 비목표

- 인사/급여/복무 승인 시스템 전체를 대체하지 않는다.
- 공문 결재, 전자문서 유통, 민원 접수 원장을 직접 대체하지 않는다.
- 챗봇 답변을 법적 판단이나 최종 민원 답변으로 자동 확정하지 않는다.
- 언론사 기사 전문을 무단 저장하거나 재배포하지 않는다.

## 5. 사용자 유형

| 사용자 | 역할 | 주요 니즈 |
| --- | --- | --- |
| 팀원 | 본인 일정 등록, 파일 자동화 사용, 민원 초안 작성 | 빠른 입력, 실수 방지, 반복업무 감소 |
| 팀장 | 팀 일정 확인, 업무 공백 파악, 민원 응대 품질 확인 | 전체 현황, 승인/검토 흐름, 감사 추적 |
| 행정지원 담당자 | 엑셀 취합, 뉴스 모니터링, 매뉴얼 관리 | 대량 처리, 예약 실행, 오류 원인 확인 |
| 시스템 관리자 | 사용자/권한/보안/운영 관리 | 접근 제어, 로그, 백업, 장애 대응 |

## 6. 핵심 기능 요구사항

### 6.1 팀원 스케쥴 관리

#### 사용자 스토리

- 팀원은 휴가, 근무, 출장, 교육, 재택 등 일정을 등록하고 수정할 수 있다.
- 팀장은 월/주/일 단위로 팀 일정과 업무 공백을 확인할 수 있다.
- 팀원은 다른 팀원의 상세 사유 중 민감 정보는 권한에 따라 제한적으로 볼 수 있다.

#### 기능 요구사항

- 캘린더 월간/주간/일간 보기 제공
- 일정 유형: 휴가, 근무, 출장, 교육, 재택, 기타
- 일정 필드: 제목, 유형, 시작일시, 종료일시, 종일 여부, 장소, 설명, 공개 범위, 작성자
- 팀/부서/개인 필터 제공
- 겹치는 일정 및 팀 최소 근무 인원 경고
- 반복 일정 등록
- 일정 변경 이력 저장
- iCal 또는 CSV 내보내기
- 향후 인사/복무 시스템 연동을 고려한 외부 일정 ID 필드 보유

#### 권한

- 본인 일정: 생성, 수정, 삭제 가능
- 팀 일정: 조회 가능
- 팀장/관리자: 팀 일정 수정 또는 숨김 처리 가능
- 민감 사유: 본인, 팀장, 관리자만 조회 가능

#### 수용 기준

- 팀원이 휴가 일정을 등록하면 팀 캘린더에 즉시 표시된다.
- 월간 보기에서 일정 유형별 색상이 구분된다.
- 권한이 없는 사용자는 비공개 일정의 상세 설명을 볼 수 없다.
- 일정 변경 시 누가 언제 어떤 값을 바꿨는지 이력이 남는다.

### 6.2 엑셀 업무 자동화

#### 사용자 스토리

- 사용자는 엑셀 파일을 업로드하고 특정 컬럼 기준으로 파일을 나눌 수 있다.
- 사용자는 여러 엑셀 파일을 같은 컬럼 구조 기준으로 병합할 수 있다.
- 사용자는 처리 결과를 다운로드하고 오류 행을 확인할 수 있다.

#### 기능 요구사항

- 지원 형식: `.xlsx`, `.xls`, `.csv`
- 파일 크기 제한 및 행 수 제한 설정
- 컬럼 자동 감지 및 미리보기
- 특정 컬럼 값 기준 파일 분리
- 여러 파일 병합
- 병합 시 컬럼 매핑, 누락 컬럼 처리, 중복 행 처리 옵션 제공
- 결과 파일 다운로드
- 처리 로그 및 오류 리포트 제공
- 원본 파일 자동 삭제 정책 설정

#### 처리 옵션

| 작업 | 옵션 |
| --- | --- |
| 분리 | 기준 컬럼, 빈 값 처리, 파일명 규칙, 시트별 처리 여부 |
| 병합 | 컬럼 정렬 기준, 중복 제거 기준, 원본 파일명 컬럼 추가 여부 |
| 검증 | 필수 컬럼 확인, 날짜/숫자 형식 검사, 행 개수 비교 |

#### 수용 기준

- 사용자가 기준 컬럼을 선택하면 컬럼 값별로 분리된 파일이 생성된다.
- 병합 결과는 원본 파일 수, 총 행 수, 오류 행 수를 함께 제공한다.
- 처리 실패 시 실패 사유와 복구 방법이 사용자에게 표시된다.

### 6.3 민원 대응 챗봇

#### 사용자 스토리

- 사용자는 민원 내용을 입력하고 관련 매뉴얼을 바탕으로 응대 방향을 추천받는다.
- 사용자는 전화, 방문, 서면, 문자 등 채널별 응대 스크립트를 생성할 수 있다.
- 담당자는 챗봇 답변의 근거 문서와 주의사항을 확인할 수 있다.

#### 기능 요구사항

- 민원 매뉴얼 파일 업로드 및 문서화
- 매뉴얼 검색 기반 답변 생성
- 답변에 근거 문서명, 조항, 페이지 또는 섹션 표시
- 응대 유형별 스크립트 생성: 친절 안내형, 단호한 안내형, 사과/재안내형, 추가자료 요청형
- 개인정보, 민감정보, 법률 판단 표현 경고
- 답변 복사 및 저장
- 대화 이력 저장 및 관리자 검토
- 매뉴얼 버전 관리

#### 답변 원칙

- 매뉴얼 근거가 없는 내용은 추정 답변으로 확정하지 않는다.
- 법적 판단, 처분 가능성, 개인 권리 제한과 관련된 내용은 담당자 검토 문구를 포함한다.
- 민원인의 개인정보를 답변 생성 과정에 과도하게 저장하지 않는다.

#### 수용 기준

- 사용자가 민원 내용을 입력하면 관련 매뉴얼 근거와 함께 초안이 생성된다.
- 근거가 부족한 경우 “매뉴얼에서 확인되지 않음” 상태를 명확히 표시한다.
- 생성 답변에는 담당자 검토 필요 여부가 표시된다.

### 6.4 뉴스 기사 수집

#### 사용자 스토리

- 사용자는 공공행정 관련 뉴스를 매일 아침 한 화면에서 확인할 수 있다.
- 관리자는 수집 키워드와 제외 키워드를 설정할 수 있다.
- 사용자는 관심 기사를 저장하거나 팀에 공유할 수 있다.

#### 기능 요구사항

- 매일 아침 예약 수집
- 키워드 기반 검색: 공공행정, 지방자치, 민원, 예산, 복지, 안전, 정책 등
- 중복 기사 제거
- 기사 제목, 언론사, 발행일시, 요약, 원문 링크 저장
- 원문 전체 저장 대신 링크와 요약 중심 저장
- 키워드/언론사/기간 필터
- 관심 기사 북마크 및 공유
- 수집 실패 알림

#### 수용 기준

- 예약 시간에 뉴스 수집 작업이 자동 실행된다.
- 동일 기사 또는 거의 동일한 제목의 중복 기사는 하나로 묶인다.
- 기사 전문이 아닌 요약과 원문 링크가 제공된다.

## 7. 공통 요구사항

### 7.1 인증 및 권한

- 사용자 로그인 필요
- 역할 기반 접근 제어 적용
- 권한 레벨: 일반 사용자, 팀장, 관리자
- 팀/부서 단위 데이터 접근 제한

### 7.2 감사 로그

- 일정 변경, 파일 업로드/다운로드, 챗봇 답변 생성, 뉴스 수집 설정 변경은 로그를 남긴다.
- 로그 필드: 사용자 ID, 작업 유형, 대상 ID, 작업 시각, IP, 성공/실패, 오류 메시지

### 7.3 개인정보 및 보안

- 민원 내용과 첨부 파일은 최소 보관 원칙 적용
- 업로드 파일은 백신/확장자/크기 검증 후 처리
- 저장 데이터 암호화 검토
- 운영 환경에서는 HTTPS 적용
- 관리자 기능은 별도 권한과 접근 로그 필요

### 7.4 접근성

- 키보드 탐색 가능
- 주요 버튼과 입력 항목에 명확한 라벨 제공
- 색상만으로 상태를 구분하지 않음
- 공공 웹 접근성 기준을 고려한 명도 대비 적용

## 8. 우선순위

| 단계 | 범위 |
| --- | --- |
| MVP | 일정 캘린더, 엑셀 분리/병합, 매뉴얼 기반 챗봇 초안, 뉴스 수집 목록 |
| 2차 | 권한 고도화, 감사 로그, 매뉴얼 버전 관리, 예약 수집 관리 |
| 3차 | 외부 시스템 연동, 대시보드, 통계, 알림, 모바일 최적화 |

## 9. 성공 지표

- 엑셀 처리 업무 평균 소요 시간 50% 이상 감소
- 팀 일정 누락/중복 확인 문의 감소
- 민원 답변 초안 작성 시간 30% 이상 감소
- 뉴스 모니터링 누락 건수 감소
- 월간 활성 사용자 및 기능별 재사용률 증가

| --- | --- | --- |
| 민원 챗봇 오답 | 민원 품질 저하, 법적 리스크 | 근거 표시, 담당자 검토, 금지 표현 필터 |
| 개인정보 포함 파일 업로드 | 보안 사고 | 보관 기간 제한, 접근 제어, 로그, 마스킹 |
| 뉴스 수집 저작권 | 재배포 리스크 | 전문 저장 금지, 링크/요약 중심 |
| 엑셀 대용량 처리 실패 | 업무 중단 | 비동기 처리, 파일 제한, 오류 리포트 |
| 일정 정보 과다 공개 | 내부 정보 노출 | 공개 범위, 권한, 비공개 필드 |
